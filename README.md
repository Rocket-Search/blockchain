    <article class="markdown-body entry-content p-3 p-md-6" itemprop="text"><h1><a id="user-content-blockchain" class="anchor" aria-hidden="true" href="#blockchain"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>blockchain</h1>
<p>blockchain;  Distributed Ledger List (SAP ERP 6.0 FI Connector)</p>
<p>first "read the manual" .</p>
<p>Than just copy the bc_server to your desired folder an start it.</p>
<p>you can access and generate blocks to the chain with the "blockchain" binary .</p>
<p>If you want to connect a SAP System to the blockchain network, implement the
"sap_bcc" binary to your SAP System.</p>
<p>if you need help , don't hesitate to contact me :
<a href="mailto:software.moore@gmail.com">software.moore@gmail.com</a></p>
<p>You can also download it at <a href="http://rocketsearch.ddns.net:8081/bc.tar.gz" rel="nofollow">http://rocketsearch.ddns.net:8081/bc.tar.gz</a></p>
<p>More Information:
<a href="http://rocketsearch.ddns.net/index.php/en/blockchain" rel="nofollow">http://rocketsearch.ddns.net/index.php/en/blockchain</a></p>
<p>##################</p>
<p>technical overview</p>
<p>All software components are written in C++ compiled with g++-7 (SUSE Linux) 7.3.1 20180307 [gcc-7-branch revision 258314]</p>
<p>Blockchain Generator:
The raw data will be transmitted via ARGV to the blockchain genereator.
Further the exact timestamp will be keept up to the nanosecond range (tv_nsec)
Than the next free block number in the chain will be determined. I use the TCP socket API with
TLS (LibreSSL) encryption for the block number determintaion in the chain peer-to-peer network.
The determintaion of the block number will also check the validation of the last block and in
the same time the prevoius hash will be requested.
From the transaction raw data, plus timestamp, plus blocknumber, plus previous hash
the transaction hash will be generated. It consits by a SHA512 hash provide by the SSL API (LibreSSL).
Than the hash will be encrypted with the GnuPG API (gpgme) and also a GPG signture will be created.
From this it is possible to determine which server has generated the block and the hash of the block is cryptographically secured against manipulation .
The GPG signature consists of an AES256 key that generates a SHA512 long signature (base64 format).
After the generation of the new block, the block is sent to the blockchain server .
The transmission uses the TLS transport layer encryption . The encryption use the following
algorithm "ECDHE-RSA-AES256-GCM-SHA512" and "Perfect Forward Secrecy" based on the TLS protocoll 1.2 .
The server certificate has a keys size from 16384 bytes .
After the block has been sent to the blockchain server, it is sent (also using TLS) to the available peers .
At leats 5 peers must confirm the vaildaitaion of the block and when they give the "OK" the block will be
included in the chain .</p>
<p>Blockchain Server:
The blockchain server checks when starting whether all the peers known to him are available and whether the various peers are valid, or
the blockers of the peers match the chain of the local server .
The communication was encrypted using TLS 1.2 "ECDHE-RSA-AES256-GCM-SHA512".
The server certificate has a 16384 bytes long key.
The blocks are stored in the in-memory NoSQL Database redis.</p>
<p>SAP Blockchain Connector:
The SAP_BC stores the data encrypted with GPG (AES256) locally on a RAM disk.
Via SSH, the SAP_BC can address the blockchain generator and send the data encrypted over the network.
therefore a user authentication takes place at Blockchain Gernertaor.</p>
<p>Features:</p>
<p>Network API</p>
<p>SHA512 transaction hash</p>
<p>GPG encyrption ang signature with AES256 and SHA512 encryption, for each block</p>
<p>Network transport encryption with TLS 1.2 and ECDHE-RSA-AES256-GCM-SHA512</p>
<p>SAP system Blockchain Connector for SAP ERP 6.0 (FI_CO etc.)</p>
<p>Timestamp for the transaction in the nanosecond range</p>
<p>Using redis in-memory database</p>
<p>Using RAM disks</p>
<p>Distribute the blocks in a Perr-to-Peer network</p>
<p>Thread based TCP connection management</p>
<p>Cyclic validation of the whole chain on all peers.</p>
